# 실시간 Q&A 웹서비스 (Slido 스타일) PRD

## 프로젝트 개요
행사 및 수업에서 참가자들이 실시간으로 질문하고 투표할 수 있는 웹 기반 Q&A 플랫폼.

## 목적
- 행사/수업 중 실시간 상호작용 향상
- 참가자의 질문을 효율적으로 관리
- 발표자가 인기 있는 질문 우선 처리

## 사용자
- 발표자: 세션 생성, 질문 관리 (숨김, 고정, 답변완료)
- 참가자: 세션 참가, 질문 작성, 좋아요 투표

## 핵심 기능

### 1. 세션 관리
- 발표자가 새로운 세션 생성
- QR 코드 또는 세션 ID로 참가자 입장
- 세션 상태 관리 (활성, 종료)

### 2. 질문 관리
- 참가자가 질문 작성
- 질문 목록 실시간 표시
- 질문에 대한 좋아요 투표 (실시간 반영)
- 발표자의 질문 상태 관리:
  * 숨김 (Hide): 질문을 목록에서 숨김
  * 고정 (Pin): 질문을 상단에 고정
  * 답변완료 (Mark as Answered): 답변 완료 표시

### 3. 실시간 기능
- Supabase Realtime을 사용한 실시간 업데이트
- 새 질문 추가 시 모든 사용자에게 즉시 반영
- 좋아요 수 실시간 업데이트
- 질문 상태 변경 즉시 반영

### 4. 사용자 인터페이스
- 발표자 대시보드: 세션 관리, 질문 목록, 상태 변경
- 참가자 뷰: 세션 입장, 질문 작성, 투표, 활성 질문 목록

## 기술 스택
- Frontend: React + TypeScript + Vite
- UI 컴포넌트: shadcn/ui
- 백엔드/DB: Supabase (인증, DB, Realtime)
- 상태 관리: React Hooks / Context API
- 배포: Vercel

## 디자인 원칙
- shadcn/ui 컴포넌트 적극 활용
- 클린하고 직관적인 UI
- 반응형 디자인 (모바일 고려)
- 접근성 고려

## 개발 전략
- 가장 간단한 방법으로 시작
- 모듈식 컴포넌트 구조
- 터미널 명령어를 통한 shadcn/ui 컴포넌트 추가

## 데이터베이스 스키마

### sessions 테이블
- id (PK)
- presenter_id (FK)
- session_code (문자열, QR 코드용)
- title (제목)
- status (active, closed)
- created_at
- updated_at

### questions 테이블
- id (PK)
- session_id (FK)
- participant_id (FK)
- content (질문 내용)
- likes_count (좋아요 수)
- status (normal, hidden, pinned, answered)
- created_at
- updated_at

### likes 테이블
- id (PK)
- question_id (FK)
- participant_id (FK)
- created_at
